🧠 🔷 SYSTEM FLOW SUMMARY
The entire app and website must support both Bangla and English language viewing options.
👤 🆕 New User Flow
Admin first adds the user using Army Number + Date of Birth
User → completes Registration
After login → an empty form will appear
User fills in their personal data
The data → will be synced to the website (roster) in real-time
👤 🔁 Existing User Flow
User only completes Registration
After login →
❌ No empty form will appear
✔ Existing data will be automatically loaded (synced from roster.html)
User can directly view their profile data
🔐 🔷 ROLE-BASED ACCESS SYSTEM
👑 Super Admin
	✔ Full control (everything)
		All user data:
		create / update / delete
		All edit requests:
		approve / reject (all type data Class)
		Can delete any user
		Role assignment:
		admin / subadmin / superadmin
		Password reset (all users)
		Access to all company data

		🌐 Website (roster):
		Can view all data
		Each section will have an Edit button
		Can modify all fields from both index & roster
		📱 Changes → will reflect instantly in the app (real-time)

🛡️ Admin (Company Based)
	✔ Limited control (own company only)
		Own company data:
		add / update / modify
		Own company users:
		approve / reject requests (all type data Class)
		reset passwords of own company members

		🌐 Website (roster):
		Own company members → full access (with edit buttons on each section)
		Other companies → only search / view access
		📱 App:
		Can edit own profile → no approval required
		Changes → will reflect instantly in the app (real-time)
👁️ Sub Admin
	✔ View-only role
	🌐 Website (roster):
	Can view all member data
	❌ Cannot:
		edit
		delete
		approve
	📱 App: Will behave as a normal member

🔗 🔷 PLATFORM STRUCTURE
📱 index.html → App (User Panel)
🌐 roster.html → Website (Admin Panel / Roster View)
⚙️ 🔷 DATA FLOW
App (index.html)
        ↓
Firestore Database
        ↓
Website (roster.html)
        ↑
        └────────────── (bi-directional flow)

👉 Data flow will be bi-directional

👉 All updates will be real-time synced

✅ FINAL UNDERSTANDING

✔ New user → fills form → data is created
✔ Existing user → auto sync → no form
✔ Admin system → company-based control
✔ Super admin → full system control
✔ Sub admin → read-only
✔ Website → restricted access (only admins)
